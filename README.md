# app-templates

> **Note on naming:** This project is in transition from `sixnet` to `0-net`.
> Throughout this document, new names are used with the current name in
> parentheses where relevant. References will be updated as the rename progresses.

App templates for the [0-net](https://github.com/0-net) self-hosted platform.

Each template is a ready-to-deploy application definition that integrates with
the [0-net core stack](https://github.com/0-net/core-stack) (`six.net`) — a self-hosted
VPN and identity platform, with SSL handling, SSO support, end-user clients
(macOS only at the moment) and complete and full open-source, including all backend
admin service dashboards needed.

The VPN itself, and SSO and SSL handling is built entirely on proven open-source tools
including ZeroTier, Authentik, and Caddy.

DNS (and DDNS) is plugable, and currently supports AWS ROUTE53 automation or a
manual guided walk-through. IP ingest, internet exposure is also plugable, and
currently provides Fritz BOX port-forwarding for home networks, or frp
(https://github.com/fatedier/frp) to tunnel inside and behind coporate
firewalls and NAT settings. 

## What's in a template

Each app directory contains:

| File | Purpose |
|---|---|
| `app.conf` | App identity, access mode, SSO config, deployment variables |
| `docker-compose.yml` | Container definition |
| `caddy.snippet` | Reverse proxy config (injected into core Caddyfile at deploy time) |
| `authentik-provider.yaml.template` | Authentik OAuth2/OIDC blueprint (SSO apps only) |
| `secrets.env.example` | Template for generated secrets |
| `setup-guide.sh` | Interactive post-deploy walkthrough |
| `README.md` | App-specific notes, gotchas, and SSO transition steps |

## Adding an app to a deployment

```bash
# 1. Add the app name to APPS= in your deployment.env
# 2. Initialise — scaffolds secrets and deployment config
make -f core/Makefile init-app APP=<name> DEPLOYMENT=.deployments/<name>/deployment.env

# 3. Deploy
make -f core/Makefile deploy-app APP=<name> DEPLOYMENT=.deployments/<name>/deployment.env
```

## App contract

### `app.conf` required fields

```bash
APP_NAME=<name>           # Must match directory name
APP_SUBDOMAIN=<prefix>    # Subdomain: <prefix>.<domain>
APP_ACCESS=vpn|public     # vpn: DNS-01 TLS + ZT IP filter; public: HTTP-01 TLS, no filter
APP_SSO=true|false        # Whether to auto-generate Authentik OAuth2 credentials
```

### Access modes

- **`vpn`** — TLS via Route 53 DNS-01 challenge, IP-filtered to ZeroTier subnet. App is
  reachable only from inside the VPN.
- **`public`** — TLS via HTTP-01 challenge (requires port 80 reachable). No IP filtering.

### Networks

Apps connect to the `0-net` (`sixnet`) Docker network (external, created by the core stack) so
Caddy can reach them by container name. Internal app networks (db, redis, etc.) are
defined per-app and are the app's own concern.

### SSO

Apps with `APP_SSO=true` get an Authentik OAuth2 provider automatically provisioned via
blueprint at deploy time. Credentials are generated by `init-app` and injected into both
the app's `secrets.env` and Authentik's `secrets.env`.

## Relation to 0-net (six.net) core

The core stack references this repo via `APPS_DIR` (default: `../app-templates` relative
to `core/Makefile`). Clone this repo next to `0-net/core-stack` or symlink it at
`0-net/core-stack/app-templates/`.

```
~/projects/0-net/app-templates/     # this repo
~/projects/0-net/core-stack/        # core stack
~/projects/0-net/core-stack/app-templates -> ../app-templates  # symlink
```
